Développement de l'application

Cette page explique les principales parties techniques du projet Risk : le graphe, les cartes, les missions, le bot et le déroulement d’un tour.

Technologies utilisées

🐍 Python

Python est utilisé pour programmer la logique principale du jeu : les joueurs, les tours, les cartes, les missions et les combats.

🌐 NetworkX

NetworkX permet de représenter la carte du jeu sous forme de graphe, avec des territoires connectés entre eux.

🎮 Pygame

Pygame est utilisé pour créer l’interface graphique du jeu, afficher la carte et gérer les interactions avec la souris.

🧱 HTML / CSS

HTML et CSS servent à construire le site web statique et à lui donner une charte graphique cohérente.

⚡ JavaScript

JavaScript ajoute de l’interactivité au site, par exemple pour afficher des informations sur les cartes.

Carte du jeu avec un graphe

La carte du jeu est représentée avec NetworkX. Chaque territoire correspond à un nœud du graphe, et chaque frontière entre deux territoires correspond à une arête. Cette structure permet de vérifier facilement si une attaque est possible.

Fonctionnement du bot

🧠 Analyse

Le bot observe les territoires qu’il contrôle et les ennemis voisins.

🛡️ Renforts

Il place ses troupes sur les territoires stratégiques.

⚔️ Attaque

Il attaque les territoires faibles pour agrandir son empire.

Déroulement d’un tour

1. Renforts

Le joueur reçoit des armées selon le nombre de territoires possédés.

2. Attaque

Le joueur peut attaquer un territoire voisin contrôlé par un adversaire.

3. Conquête

Si l’attaquant gagne, il prend le contrôle du territoire ennemi.

4. Carte bonus

Si le joueur conquiert au moins un territoire, il peut recevoir une carte.
